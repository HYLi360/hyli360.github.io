## Question.

我新写了一个 Python 程序包！这个程序包可以从终端调用，其中一个功能是将 `.gz` 文件重新压缩为 `.bgz` 文件，方便随机读取。

不过随着程序包越做越复杂，我需要一个全局配置文件来持久化程序配置，例如使用的语言、部分软件的文件路径，等等。我如何能够持久化配置？最适合放置这种持久化配置文件的地方在哪里？

## Answer.

首先回答持久化配置该放哪里的问题。

**配置文件绝对不能放在软件包所在目录（`side-packages`）！**这样做会造成两个问题：

- 如果谁卸载了这个软件包，那么配置文件就会随着这个软件包一起删掉。下次再进行安装时，就需要从头开始。
- 极端情况下，这个路径可能不可写（例如 `usr/lib`，如果要在这里写入配置文件，就需要额外给一次管理员权限，虽然多数情况下不会使用这里的 Python 环境，而是自己新建的虚拟环境）。

合适的做法是，放在系统的 home 目录（POSIX 的 `/home/user_name/.config/myapp`，或者 Windows 上的 `%APPDATA%\myapp`）。——这就牵涉到第二个问题：如何考虑系统差异性，同时实现在两种系统上持久化配置？

介绍一下 `os.name`——它的功能就是检查其所在系统为 Windows（`nt`）还是 POSIX（`posix`）。其结果也有可能是 `java`，但在上下文均未提到 Jython 的条件下，不考虑这个情形。

```py
import os

# os.environ 解析环境变量
# 而 os.path.expanduser 将原路径上的 ~ 替换为该用户的 home 目录
HOME_PATH = os.environ["APPDATA"] if os.name == "nt" else os.path.expanduser("~/.config")
```

> `sys.platform` 具有更细的识别粒度。不过这里我们只需要区分这两种平台即可，所以使用了 `os.name` 而不是 `sys.platform.system()`。

接下来就是写入或修改配置文件。配置文件的格式种类很多，包括 JSON、TOML，等等等等；TOML 格式很简洁好读，但 Python 内置的 `tomllib`（Python 3.11+）只知道如何读取，而不明白如何写入；而 JSON 可通过内置库 `json` 读写，读取结果则是我们相当熟悉的嵌套字典。

显然，JSON 是个不错的选择。不过如果有直接对配置文件进行操作的需求，选择 TOML 也无妨——想完全采用 TOML 时，可加装 `tomli-w` 第三方模块。不过在这里我选择把它藏起来，之后通过 `myapp` 软件包 `config` 模块，间接进行控制，这种情形显然没有“人类可读性”需求，使用 JSON 完全没问题。

```py
import json
import os
from pathlib import Path

# 上一步的“获取 home 目录”
HOME_PATH = os.environ["APPDATA"] if os.name == "nt" else os.path.expanduser("~/.config")

# 使用 mkdir 可自动生成中间目录
Path(HOME_PATH / "myapp").mkdir(exist_ok=True)

# 此处是默认配置
CONFIG_DEFAULT = {"lang": "en"}

# 只在安装时执行一次
# 如果发现文件存在，则也不执行
if not Path(HOME_PATH / "myapp" / "config.json").exists():
	json.dump(CONFIG_DEFAULT, open(f"{HOME_PATH / "myapp" / "config.json"}", 'w'))
```
